# Creating your next course

This topic is intentionally seeded as frontier territory. It has a title and a stable path, but no `entered_at` timestamp and no journal entries yet.

When you open it, the mentor should treat it as unexplored territory and help you decide what the next real course should be.

After this tutorial, make a course for something real. Pick a topic where repeated teaching turns would help more than a single answer.

Start with the smallest useful course frame:

- Title: "Databases from first principles"
- Description: "Teach me indexes, transactions, and query planning with small examples."
- Attached folder: optional, useful when the course should reference a project.

If the shape is unclear, give the mentor a concrete seed:

> I want to understand OAuth well enough to implement it safely in a small app.

Keep the first course small enough to finish. Let the topic map emerge from teaching turns instead of trying to perfect it up front.
